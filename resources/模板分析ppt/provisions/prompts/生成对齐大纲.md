# 生成对齐大纲

你是一位演示文稿规划专家。根据已确定的模板计划，将演示大纲与模板计划**严格 1:1 对齐**，并填充具体内容细节。

## 输入

### 演示大纲 `{演示大纲}`
<presentation_outline>
$input_1
</presentation_outline>

自动生成的演示文稿结构，包含：
- `title`, `subtitle`: 演示标题
- `sections`: 幻灯片列表，每个包含 slide_type、topic、content_hints
- `narrative_flow`: 整体叙事流程

### 模板计划 `{模板计划}`
<template_plan>
$input_2
</template_plan>

已确定的模板使用计划，包含：
- `plan`: 页面索引序列，如 `[0, 1, 3, 3, 5, 14]`
- `mapping`: 每个位置的详细映射信息

### 简化概览 `{简化概览}`
<template_overview>
$input_3
</template_overview>

模板各页面的能力信息，包含每页的 `text_slots` 等。

### 所有内容参考 `{所有内容参考}`
<content_references>
$input_4
</content_references>

领域知识材料，用于丰富具体内容细节。

### 需求分析 `{需求分析}`
<requirements_analysis>
$input_5
</requirements_analysis>

演示需求分析结果，用于调整内容深度和风格。

## 任务

生成一个**对齐大纲**，满足以下约束：

### ⚠️ 核心约束

```
len(对齐大纲.sections) == len(模板计划.plan)
```

对齐大纲的 section 数量**必须等于**模板计划的页面数量。

### 对齐逻辑

| 情况 | 处理方式 |
|------|---------|
| 演示大纲有 N 个 sections | 确保输出也是 N 个 sections |
| 模板页面复用（如第3页用两次）| sections 对应位置保持独立内容 |
| 模板页面 text_slots 数量 | content_plan 的 blocks 数量要匹配 |

### 内容丰富化

关键任务是将 `content_hints`（简略提示）转化为 `content_plan`（详细计划）：

1. **从 {所有内容参考} 提取具体数据**
   - 数字、事实、案例
   - 专业术语、核心概念
   - 具体描述和细节

2. **根据 {需求分析} 调整风格**
   - `topic_depth`: 深入 → 详细技术细节；概述 → 简洁要点
   - `audience_level`: 专家 → 专业术语；初学者 → 通俗解释
   - `tone`: 正式/轻松/激励

3. **匹配模板页面 text_slots**
   - 查看 {简化概览} 中对应页面的 slots 数量和角色
   - content_plan 的 blocks 数量应与 slots 匹配

### 每个 Section 的内容

每个 section 需要包含：
1. **slide_index**: 在最终演示中的位置（0-based）
2. **template_page**: 使用的模板页面索引
3. **page_type**: 页面类型
4. **topic**: 页面主题
5. **content_plan**: 该页需要填充的具体内容

`content_plan` 结构要与模板页面的 `text_slots` 对应：
- 如果页面有 `主标题` 和 `副标题` slots → content_plan 有 `title` 和 `subtitle`
- 如果页面有 3 个内容块 → content_plan 有 `blocks` 数组，每块有 `title` 和 `description`

1. **严格对齐**: section 数量必须等于 plan 长度，这是最重要的约束
2. **内容丰富**: 使用内容参考填充具体数据，不要只是抽象描述
3. **风格一致**: 根据需求分析调整专业程度和语言风格
4. **结构匹配**: content_plan 的 blocks 数量要匹配目标页面的 text_slots
5. **topic 明确**: 每个 section 要有清晰的 topic 描述

**重要：** 你的响应必须是有效的JSON，且必须包含这两个顶级键（thinking 和 result）。
